# Local interactive and JSONL harness boundary

## Decision question

What public boundary and capability surface does the Lucy harness expose to interactive users and machine clients?

## Current decision

Lucy MUST run as a local macOS/Linux process and MUST retain its newline-delimited JSON machine protocol. When both standard input and standard output are terminals, an invocation without a mode flag MUST start the TUI. When either stream is not a terminal, the invocation MUST use JSONL automatically. `--jsonl` MUST force JSONL and `--tui` MUST force the interactive frontend; the latter requires a usable terminal. The TUI is a frontend over the same normalized event and turn engine, not a new provider or tool boundary. Its slash picker MUST combine discovered skill names with Lucy-owned `/settings` and `/exit` commands without persisting or expanding those commands as skills. `/settings` MUST ignore trailing arguments and open the idle-only settings menu; `/exit` MUST terminate an idle TUI session. The TUI MUST render each background worker as a compact one-line entry in a list between the message input and the bottom status line, showing task id and a short task preview. When the input has focus, Down from the last visual input row MUST move focus into that list when it is non-empty; Up from the first list row or Esc MUST return focus to the input. While a list entry is focused, the TUI MUST show that worker's live message stream in a borderless overlay with white text and square corners immediately above the input at the same slot used by the slash skill picker. Worker stream observation is a TUI-internal frontend signal: it MUST NOT add public JSONL events. The TUI transcript MUST suppress cards for `spawn_subagent`, `check_subagent`, `wait_subagent`, `send_subagent`, and `cancel_subagent` while retaining their normalized JSONL events and persisted model observations. A successful `spawn_subagent` is represented by its running-list row. For a running target worker, `check_subagent` and `send_subagent` MUST briefly flash that row, `wait_subagent` MUST show `Waiting for` with a spinner while pending, and `cancel_subagent` MUST show `Cancelling` with a spinner until the worker reaches its terminal lifecycle transition. A failed or unknown-target lifecycle tool result MUST remain a TUI transcript error. After an active TUI turn reaches the worker-finished boundary and `busy` is released, the TUI MUST emit a terminal-native OSC 777 desktop notification for normal completion, interruption, or error; terminals without support may ignore this frontend signal. It MUST NOT be emitted in JSONL mode, and a notification write failure MUST NOT change turn behavior.

Lucy MUST NOT be a network service in v1. The harness MUST expose `cmd`, `spawn_subagent`, `check_subagent`, `wait_subagent`, `send_subagent`, and `cancel_subagent` as its only model tools; it MUST NOT provide built-in `read`, `write`, `edit`, or other file-operation tools. `spawn_subagent` accepts only a task. It creates an isolated, one-level worker with the session's model, reasoning effort, immutable boot context, and cwd but no parent conversation or tool output; callers MUST NOT override the worker model or effort. A worker can use `cmd` but MUST NOT receive `spawn_subagent`. Up to four workers run concurrently across the process. `spawn_subagent` returns a task ID and queued status immediately rather than waiting for the worker. Every spawned worker is attached to the logical main turn that created it. If the model otherwise completes while an attached worker is running, the harness MUST suspend that turn without emitting `turn_end`; a terminal child result MUST resume the same logical turn through a harness-owned background-result observation at the next safe provider boundary. A child completion MUST NOT be represented as user input or independently start a new user turn. User messages accepted while a turn is active or suspended are queued behind ready child results. Parent-turn cancellation cancels only the active turn, does not cancel its workers, and prevents their later completion from reviving the interrupted turn; those results remain pending for the next user-driven turn. Process shutdown cancels all active workers and their commands. Pi-style Agent Skills are input-context packages, not additional model tools: Lucy MAY discover their metadata at new-session boot and expand an explicit `/<name> [args]` user message into that skill's saved `SKILL.md` content, but it MUST NOT expose a skill tool or execute a skill itself. The LLM integration MUST retain the OpenAI-compatible Chat Completions adapter for OpenRouter and MAY provide explicitly bounded provider adapters for authenticated provider-specific APIs. The Codex subscription adapter MUST target the Codex Responses endpoint, translate its stream into Lucy-owned normalized events, and keep its endpoint and authentication contract out of generic `[llm]` configuration.

Provider SSE and tool-call chunks MUST be converted into Lucy-owned normalized events. Provider-specific response chunks MUST NOT become the public JSONL protocol or TUI output. JSONL mode MUST emit only newline-delimited events on stdout and diagnostics to stderr. The TUI MUST render the same normalized event sequence, including streamed assistant deltas, tool calls/results, errors, normal turn completion, and user interruption. One process handles one active turn at a time.
Lucy MUST NOT impose a fixed count or provider-round limit on model tool calls within an active main-agent turn or spawned worker. Resource bounds remain in force for provider SSE bodies, tool-call fields and arguments, command execution time/output, concurrent spawned workers, cancellation, and process shutdown.

A spawned worker MUST be represented as a first-class child session linked to its parent session by `parent_session_id`. The child MUST persist its full secret-redacted transcript and terminal status in a separate append-only JSONL session file while retaining the one-process, same-cwd execution model. Child sessions MUST use only the boot snapshot plus their delegated task as initial model context; they MUST NOT inherit the parent transcript or tool output. The first implementation MUST NOT resume a running child after Lucy restarts, but MUST record process-shutdown interruption before graceful process exit when possible.

The harness MUST expose child lifecycle control for `wait_subagent`, `send_subagent`, and `cancel_subagent` in addition to `spawn_subagent` and `check_subagent`. `send_subagent` MUST queue input for the child's next safe provider boundary without interrupting an in-flight provider stream or command. `cancel_subagent` MUST cancel provider and command work at their nearest safe cancellation point. `wait_subagent` MUST wait for a terminal child state without canceling the child when its wait timeout expires.

A completed child notification MUST NOT interrupt an active main provider stream, command, or tool-call batch. The parent session MUST first persist a typed pending background result containing a unique completion ID, task and child-session identity, terminal status, result, and completion time. After the current provider response's complete tool-call batch and results are persisted, the harness MUST drain all ready child completions in completion order before the next provider request. It MUST commit each result to the main context as a harness-owned synthetic tool observation and then persist one delivered record that references the completion ID and logical main turn. Pending and delivered lifecycle transitions MUST be append-only, secret-safe, visible in the normalized JSONL protocol and TUI transcript, and reconstructible on resume. The provider adapter MAY materialize a delivered result as a synthetic assistant tool call followed by its matching tool result, but these provider messages MUST NOT be the persistent source of truth and the internal completion tool MUST NOT be model-callable. A completion is delivered when it is committed to the main context, not when a provider response succeeds, and MUST be committed at most once. An explicit `wait_subagent` result counts as delivery and MUST suppress later synthetic delivery of the same completion. Ready child results take precedence over queued user messages at this safe boundary.

The TUI background-worker list MUST contain only currently running workers. A terminal worker MUST leave the list as soon as its terminal state and pending result are persisted, independently of whether the main agent has consumed that result. Pending and delivered transitions remain visible as transcript history. When several attached workers finish at different times, the first terminal result resumes the suspended main turn with every result ready at that boundary; if other workers remain after the model responds, the turn MAY suspend and resume again. `turn_end` MUST be emitted exactly once, only after no attached worker remains running and the model completes. A canceled, failed, or interrupted child follows the same result-delivery path as a completed child.

Automatic context compaction MUST run only at a safe boundary inside that active turn: after a provider response and all associated `cmd` results are complete and persisted, and before the next provider request. It MUST NOT interrupt an in-flight provider SSE stream or execute tools from the compaction-summary request. Compaction starts when the estimated context reaches at least 95% of the model window, uses the configured model with no tools, and continues the original turn only after a successful summary and context replacement. A failed or canceled compaction MUST NOT persist a summary or replacement boundary and MUST NOT emit normal turn completion.

ESC cancellation MUST stop a provider stream or running `cmd` at the nearest safe cancellation point. A canceled command's process group MUST be terminated when possible. Cancellation MUST emit a normalized interruption event and MUST NOT emit a normal turn completion event.

## Context and forces

The goal is a thin, embeddable harness rather than a full coding-agent product. A local trusted model needs command execution and conversation state, but callers should not depend on OpenAI/OpenRouter chunk shapes. A local process boundary keeps integration simple without adding a network service or multi-tenant concerns. Explicit CLI authentication is permitted for a provider adapter and is separate from the model-facing protocol.

## Invariants

- Machine input messages and output events are LF-delimited JSON records.
- A successful turn exposes assistant deltas, normalized tool calls/results, and an explicit turn completion event.
- An interrupted turn exposes all safe events emitted before cancellation and one interruption event; it does not claim normal completion.
- A model tool call is executed by the harness before the next provider turn, except that `spawn_subagent` acknowledges queued work immediately.
- Active main-agent and spawned-worker tool loops have no fixed tool-call count or provider-round ceiling; they stop on a provider completion, cancellation, provider failure, or another existing resource boundary.
- `cmd`, `spawn_subagent`, `check_subagent`, `wait_subagent`, `send_subagent`, and `cancel_subagent` are the only model-facing tools. A spawned worker receives only `cmd`; it cannot recursively delegate.
- The harness executes no more than four spawned workers concurrently. Each worker inherits the session cwd and immutable boot context, but not parent transcript or tool output; its task is the complete delegated context.
- A spawned worker MUST use the session model and reasoning effort as well as the session provider endpoint and credential; the tool schema and argument validation MUST reject per-worker model or effort overrides. Its completion is persisted in the child session and as one typed pending parent result; it is never parent user input and never starts a separate user turn. `check_subagent`, `wait_subagent`, `send_subagent`, and `cancel_subagent` report or control the current in-process task state.
- While a provider turn is active or suspended, user messages and completed-worker results are queued. The harness never has more than one active provider request, drains ready child results serially at safe boundaries, and records at most one delivery per completion ID.
- A skill invocation is a user-message expansion, not a tool call or public protocol event. Lucy-owned slash commands are TUI controls and never enter that expansion path.
- A new session discovers skills from configured built-in locations using symlink-safe reads, catalogs only metadata for model selection, and persists the secret-redacted skill snapshot. A resumed session invokes only that snapshot and never rereads changed skill paths.
- Automatic compaction is an internal control phase, not a new public JSONL event or model-facing tool capability; TUI-only progress may use an internal frontend signal.
- TUI completion notifications are frontend-only OSC 777 signals, are emitted at most once when an active turn releases `busy`, cover completion/interruption/error outcomes, and use fixed Lucy-owned secret-safe text.
- Live background-worker streams, per-row lifecycle-tool notices, and the focused-worker stream overlay are TUI-only frontend surfaces. Normalized terminal and delivery lifecycle events remain public JSONL events, while provider-specific worker deltas remain private frontend signals. The between-input-and-status list contains running workers only.
- Parent lifecycle-tool cards for `spawn_subagent`, `check_subagent`, `wait_subagent`, `send_subagent`, and `cancel_subagent` are suppressed only from the TUI transcript; their normalized events and persisted observations remain intact. A running worker row represents successful spawn creation, check/send flash briefly, wait/cancel annotate the targeted running row, and failures or unknown targets remain transcript errors.
- Compaction never treats a partial assistant/tool-call stream as a complete context record, and its summary request has no `cmd` tool definition.
- The active provider key is not emitted in protocol events, TUI output, or diagnostics; key values that cannot be safely represented are rejected before output. TUI mode rejects keys containing fixed terminal input/border characters.
- No network listener, authentication layer, approval UI, or sandbox is required by this boundary.

Tool loops may continue for an arbitrary number of provider rounds until the model completes or an existing cancellation/resource boundary stops them. This removes an artificial count ceiling while retaining bounds on each provider payload, command, worker, and cancellation path.

## Alternatives and trade-offs

A library, HTTP server, or raw provider-stream pass-through would increase coupling or implementation surface. Additional file tools would make Lucy a larger coding agent and are intentionally left to future extensions or callers. A bounded non-recursive delegation tool permits parallel specialized work without turning the process into a network service or exposing the parent transcript to every worker. Treating skills as model tools would expand the protocol and execution surface; progressive-disclosure context plus explicit user invocation preserves the one-tool boundary while supporting reusable workflows.

## Consequences

Interactive users receive a terminal chat experience by default, while scripts retain an explicit and automatic JSONL path. Clients must implement a small event consumer, but they remain independent of provider wire formats. Skill command text works through the existing message input in both frontends and needs no protocol extension. The harness serializes one active provider request at a time while a logical turn may suspend for attached workers and resume repeatedly as results become ready. This can add provider calls for workers that finish at different times, but no child result is promoted to user authority and no separate user turn is fabricated. Compaction adds provider latency and a second model request at a safe turn boundary, but does not expose another model-callable tool. A cancellation can leave a valid partial transcript, pending child results, and interruption record rather than a normal completed turn; cancellation during compaction follows the same rule without persisting a partial summary. Supported terminal emulators may surface a fixed-text desktop notification when the TUI turn ends, while unsupported terminals and JSONL clients remain unaffected.

## Enforcement

Integration tests MUST exercise TTY and non-TTY mode selection, JSONL input/output, normalized text streaming, the cmd and subagent lifecycle tool loops, explicit skill invocation and snapshot persistence, stdout purity, and interruption ordering. Tests MUST prove that workers receive only their delegated task plus boot context, always inherit the session model/effort and reject override arguments, cannot recursively delegate, run no more than four concurrently, are acknowledged without blocking the parent, suspend and resume the same logical turn without a user message, support status inspection, and are canceled on process shutdown. Tests MUST prove that pending and delivered records survive resume, explicit wait and automatic delivery cannot duplicate one completion, simultaneous ready results are batched, terminal workers immediately leave the running list, and `turn_end` is emitted once after the final attached worker result is handled. They MUST verify normalized terminal/delivery JSONL events, that a provider-specific stream is not forwarded as a public event, that ESC cancels provider and command work within bounded behavior, and that the TUI reflects the normalized event sequence. Compaction tests MUST verify that summary requests expose no tools, occur only after complete provider/cmd/background-result boundaries, do not add unrelated public JSONL records, and leave the turn/session valid on failure or cancellation. TUI tests MUST verify slash-picker discovery and dispatch for Lucy-owned commands, settings argument handling, `/exit`, model-list viewport tracking, and advertised effort selection. They MUST verify the running-only worker list, Down/Up/Esc focus movement, focused-worker stream overlay placement, pending/delivered transcript history, suppression of parent lifecycle-tool cards, active-row check/send/wait/cancel notices, and transcript errors for failed or unknown lifecycle-tool targets. They MUST also verify that completion, interruption, and error paths each emit one fixed-text OSC 777 notification only when `busy` is finally released, and that notification write failures do not change the turn result.

## Revisit when

Reconsider this decision if callers require concurrent sessions in one process, a remote deployment, additional providers with incompatible tool protocols, first-class file operations, recursively delegated or durable workers, a first-class skill execution protocol, or a different interactive frontend boundary.
